# 图片新增简介字段 — 后端改动说明

## 概要

`picture` 表新增 `description`（图片简介）字段，涉及上传、查询、修改三个接口的变更。

---

## 1. 涉及的接口变更

### 1.1 响应体变化（所有返回 PictureVO 的接口）

所有返回图片数据的接口（上传、分页查询、修改），响应体中新增 `description` 字段：

```json
{
  "id": 1,
  "url": "https://...",
  "name": "example.jpg",
  "description": "这是一张示例图片",   // ← 新增
  "size": 102400,
  "width": 1920,
  "height": 1080,
  "contentType": "image/jpeg",
  "format": "jpg",
  "userId": 1,
  "user": { ... },
  "createTime": "2025-07-10T12:00:00"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| description | string | 图片简介，可为 null |

### 1.2 `PUT /picture/update` — 修改图片信息

**请求体变化：** 新增可选字段 `description`，同时 `name` 改为可选（但至少传一个）。

```json
{
  "id": 1,
  "name": "新名称.jpg",         // 可选，不想改可不传
  "description": "新的图片简介"  // 可选，新增
}
```

> 注意：`name` 和 `description` 至少传一个，否则返回参数错误。

### 1.3 `GET /picture/page` 和 `GET /picture/my/page` — 分页查询

新增查询参数 `description`，支持模糊搜索：

```
GET /picture/page?current=1&pageSize=10&description=简介&name=&minSize=&maxSize=
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| description | string | 否 | 图片简介（模糊匹配），新增 |

---

## 2. 前端类型更新

```typescript
// PictureVO 新增 description
interface PictureVO {
  id: number
  url: string
  name: string
  description: string | null   // ← 新增
  size: number
  width: number
  height: number
  contentType: string
  format: string
  userId: number
  user: UserVO | null
  createTime: string
}
```

```typescript
// PictureUpdateRequest 新增 description
interface PictureUpdateRequest {
  id: number
  name?: string         // 改为可选，但 id + (name | description) 至少传一个
  description?: string  // ← 新增
}
```

---

## 3. 存量数据库迁移

已部署的数据库执行以下 SQL（不会丢失数据）：

```sql
ALTER TABLE `picture`
    ADD COLUMN `description` VARCHAR(512) DEFAULT NULL COMMENT '图片简介' AFTER `height`;
```
