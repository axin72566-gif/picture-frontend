# 图片模块接口对接文档

## 基础信息

| 项目 | 说明 |
|------|------|
| Base URL | `http://localhost:8080/api` |
| Content-Type | `application/json`（上传接口为 `multipart/form-data`） |
| 鉴权方式 | 请求头 `Authorization: Bearer <token>` |

## 接口一览

| 方法 | 路径 | 说明 | 是否需要登录 |
|------|------|------|:----------:|
| POST | `/picture/upload` | 上传图片 | ✅ |
| GET | `/picture/page` | 分页查询所有图片 | ❌ |
| GET | `/picture/my/page` | 分页查询我的图片 | ✅ |
| PUT | `/picture/update` | 修改图片名称 | ✅ |
| DELETE | `/picture/delete/{id}` | 删除图片 | ✅ |

---

## 1. 上传图片

```
POST /picture/upload
Content-Type: multipart/form-data

file: (二进制文件)
```

## 2. 分页查询图片（公开）

**请求**

```
GET /picture/page?current=1&pageSize=10&name=&minSize=&maxSize=&sortField=createTime&sortOrder=desc
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| current | int | 否 | 页码，默认 1 |
| pageSize | int | 否 | 每页条数，默认 10 |
| name | string | 否 | 图片名称（模糊匹配） |
| minSize | long | 否 | 文件大小下限（字节） |
| maxSize | long | 否 | 文件大小上限（字节） |
| sortField | string | 否 | 排序字段：`name`、`size`、`createTime` |
| sortOrder | string | 否 | 排序方向：`asc`、`desc`，默认 `desc` |

**响应**

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "records": [
      {
        "id": 1,
        "url": "https://...",
        "name": "example.jpg",
        "size": 102400,
        "width": 1920,
        "height": 1080,
        "contentType": "image/jpeg",
        "format": "jpg",
        "userId": 1,
        "user": {
          "id": 1,
          "userAccount": "zhangsan",
          "userName": "张三",
          "userAvatar": "https://...",
          "userProfile": "个人简介",
          "userRole": "user"
        },
        "createTime": "2025-07-10T12:00:00"
      }
    ],
    "total": 100,
    "size": 10,
    "current": 1,
    "pages": 10
  }
}
```

> `user` 字段内的 `createTime` 和 `updateTime` 为用户注册时间和资料更新时间，非必要可不展示。

---

## 3. 分页查询我的图片（需登录）

**请求**

```
GET /picture/my/page?current=1&pageSize=10&name=&sortField=createTime&sortOrder=desc
```

参数同接口 2，但自动过滤为当前登录用户上传的图片。

**响应**

与接口 2 结构一致。

## 4. 修改图片名称

**请求**

```
PUT /picture/update
Content-Type: application/json

{
  "id": 1,
  "name": "新名称.jpg"
}
```

**响应**

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "id": 1,
    "url": "https://...",
    "name": "新名称.jpg",
    ...
    "user": { ... },
    "createTime": "..."
  }
}
```

---

## 5. 删除图片

**请求**

```
DELETE /picture/delete/1
```

**响应**

```json
{
  "code": 0,
  "message": "ok",
  "data": null
}
```

## 6. 统一响应格式

```typescript
interface BaseResponse<T> {
  code: number    // 0 表示成功，非 0 表示错误
  message: string // 提示信息
  data: T         // 响应数据
}
```

| code | 说明 |
|:----:|------|
| 0 | 成功 |
| 40000 | 请求参数错误 |
| 40100 | 未登录 |
| 40101 | 无权限 |
| 40400 | 数据不存在 |
| 50000 | 系统内部错误 |

## 7. 前端类型定义参考

```typescript
// 图片 VO
interface PictureVO {
  id: number
  url: string
  name: string
  size: number
  width: number
  height: number
  contentType: string
  format: string
  userId: number
  user: UserVO | null
  createTime: string
}

// 用户 VO（缩略版，挂在图片下的子对象）
interface UserVO {
  id: number
  userAccount: string
  userName: string
  userAvatar: string
  userProfile: string
  userRole: string
  createTime?: string
  updateTime?: string
}

// 分页响应
interface PageResult<T> {
  records: T[]
  total: number
  size: number
  current: number
  pages: number
}
```

## 8. 前端使用示例（Vue / React）

展示创建者头像和名称：

```tsx
// 假设 pictureList 为 PictureVO[]
pictureList.map(pic => (
  <div key={pic.id}>
    <img src={pic.url} alt={pic.name} />
    <p>{pic.name}</p>
    <div class="creator">
      <img src={pic.user?.userAvatar} alt="" />
      <span>{pic.user?.userName ?? '未知用户'}</span>
    </div>
  </div>
))
```

> 注意：`user` 字段可能为 `null`（极少情况，如创建者已被删除），前端需做空值判断。
